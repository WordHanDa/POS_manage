## 1. 系統概覽 (Home)

**功能：** 提供即時的經營狀態摘要。

* **今日總營收：** 自動加總今日已成交訂單的最終金額。
* **今日訂單數：** 統計今日產生的不重複訂單總量。
* **待出餐項目：** 顯示廚房目前尚未完成的出餐件數，協助掌握出菜進度。
* **自動刷新：** 系統每 60 秒會自動更新一次數據，確保資訊同步。

## 2. 品項管理 (Item Management)

**路徑：** `導覽列 > 品項管理`
**功能：** 管理餐廳提供的所有商品（如調酒、單杯酒、餐點等）。

### 新增與編輯品項

1. **填寫資訊：** 輸入名稱、價格（支援小數）、選擇種類（如 SPARKLING, CLASSIC 等）。
2. **圖片設定：** 貼上圖片路徑（如 `images/drink1.jpg`）。若圖片載入失敗，系統會自動切換為預設佔位圖。
3. **描述：** 詳述產品特色，表格中點擊描述文字可「展開/收合」查看完整內容。
4. **排序功能：** 點擊表格標題（ID、名稱、類型、價格）可進行升冪或降冪排序。

## 3. 座位佈局 (Seat Management)

**路徑：** `導覽列 > 座位管理`
**功能：** 視覺化配置餐廳內桌位分布。

### 視覺化配置

* **拖拽定位：** 右側設有 2D 平面網格畫布。您可以直接用滑鼠「拖動」藍色桌位方塊，放開後系統會自動對齊網格並立即儲存位置至資料庫。
* **快速編輯：** 點擊畫布上的桌位，左側表單會自動載入該桌資訊進行名稱或坐標微調。

## 4. 點餐流程 (Order & Details)

本模組分為「訂單列表」與「訂單明細」兩層級。

### A. 建立訂單 (ORDER.jsx)

1. **快速開單：** 選擇座位（下拉選單會顯示該座位目前是否有活躍訂單），可預填折扣金額與備註。
2. **訂單狀態：** 表格中以顏色區分「未結帳（紅色）」與「已結清（灰色）」。

### B. 明細操作 (ORDER_DETAIL.jsx)

點擊訂單列表的「明細」按鈕進入。

* **新增品項：** 支援「類別過濾」。先選類別（如：CLASSIC），品項選單將自動篩選出對應商品，加速點餐。
* **折扣調整：** 針對單一品項，若尚未出餐，可點擊「修改」調整該項目的折扣比例（%）。
* **出單按鈕：** 點擊「點擊出單」後，該品項會傳送到廚房出餐清單，且狀態變為不可移除。

## 5. 出餐管理 (Kitchen Queue)

**路徑：** `導覽列 > 出餐順序`
**功能：** 廚房專用界面，顯示待製作的品項。

* **優先級排序：** 系統根據下單時間排序，標記 P1, P2... 代表優先順序。
* **超時預警：** 若訂單等待超過 **5 分鐘** 尚未出餐，該列會變為「淡紅色背景」並顯示紅色警告文字，提醒人員優先處理。
* **確認出餐：** 製作完成後點擊「確認出餐」，該項目會標記為 DONE 並從待處理計數中移除。

---

## 6. 營業報表與稽核 (Audit)

**路徑：** `導覽列 > 營業報表`
**功能：** 會計稽核與歷史訂單查詢。

### 經營數據分析

* **日期篩選：** 可選定特定時間區間查看營收。
* **財務統計：** 自動計算該區間的「總營收」與「總折讓金額（Discounts）」。
* **PDF 報表匯出：** * 點擊任一訂單的「明細」展開。
* 點擊「匯出 PDF」，系統會產生一份包含訂單編號、詳細品項小計、備註及最終實收金額的稽核報告，方便列印存檔。

## 🛠 技術提示與維護

* **連線設定：** 本系統連接至後端 API：`https://posserver-sigma.vercel.app`。
* **資料緩存：** 系統在重要操作（如出餐、結帳）後會自動觸發 `fetch` 重新抓取最新資料。
* **瀏覽器支援：** 建議使用 Chrome 或 Edge 以獲得最佳的「拖拽座位」體驗。
